## As duas etapas

A integração tem **dois movimentos** distintos:

| Etapa | Onde roda | O que faz |
|---|---|---|
| **1. Issue** | Backend do parceiro → Backend Solvefy | Troca a API key (segredo) por um embed token de 2 min vinculado a um usuário específico. |
| **2. Exchange** | Browser do usuário (iframe) → Backend Solvefy | Troca o embed token por um par `accessToken`+`refreshToken` normal e abre o produto. |

A separação garante que **a API key nunca toca o browser** — quem entra no
iframe é só o token efêmero.

## Auto-provisioning

Na **primeira chamada** com um `externalUserId` novo, o Solvefy cria um User
internamente e o vincula ao seu Partner. **Chamadas subsequentes** com o mesmo
`externalUserId` reusam o mesmo User.

Implicações:

- Você **não precisa** chamar nenhum endpoint de cadastro antes.
- O `externalUserId` é o id estável do usuário no **seu** sistema. Use o
  mesmo valor sempre — se mudar, o Solvefy vai criar um User novo.
- O `email` e `name` do payload são gravados na primeira chamada. Atualizações
  posteriores não mudam o User existente (no MVP atual).

<Note>
  O User auto-provisionado **não tem senha** — ele só pode entrar pelo fluxo
  embed do parceiro que o criou. Não há vazamento entre parceiros.
</Note>

<Note>
  **Isolamento por usuário**: cada `externalUserId` distinto tem suas próprias
  campanhas e métricas. Usuários diferentes do mesmo parceiro **não** compartilham
  dados entre si — quem criou enxerga só o que criou. Se você quer que duas
  pessoas vejam as mesmas campanhas, mande o mesmo `externalUserId` para ambas.
</Note>

## O que o usuário vê

Ao abrir o iframe pela primeira vez:

1. Tela de loading rápida ("Autenticando…")
2. Redireciona pra `/overview?embed=1` (ou para a tela escolhida no
   parâmetro `page` — ver [Iframe (browser-side)](/integration/iframe))
3. Conteúdo do produto, com brand do parceiro no título da aba

Em navegações internas dentro do iframe, o modo embed é preservado via
`sessionStorage` — não some se o usuário clicar em links internos do
produto.

## Tempo de vida dos tokens

| Token | TTL | Onde mora |
|---|---|---|
| **Embed token** | 2 minutos | Apenas em trânsito (URL do iframe). Single-use. |
| **Access token** | 15 minutos | `localStorage` do iframe. Renovado automaticamente. |
| **Refresh token** | 7 dias | `localStorage` do iframe. Sessão silenciosamente prolongada. |
| **API key (parceiro)** | Não expira | Cofre de segredos do **seu** backend. |

Quando o usuário fecha a aba do parceiro, o `localStorage` do iframe persiste
naquele browser. Próxima vez que ele abrir, se o `refreshToken` ainda for
válido, **não precisa** de novo embed token. Quando expirar, volte ao passo 1.
